WSN - Safety Net for Developers

An automatic local safety net for your code.

What is WSN?

WSN is a lightweight, local-first background utility that watches your project directories and captures snapshots of your work as you write it. It acts as an invisible safety net, ensuring you never lose a flow-state breakthrough, a broken regex fix, or an accidental deletion again.
Even if you forgot to commit to Git.

Why use WSN?

  • Very simple to use: No manual staging, no commit messages required, no administrative workflow. No 1-hour tutorial to use it. WSN is meant to be simple to use and has only a few commands to manage your projects.
  • Granular Timeline: Unlike Git, WSN tracks your progress at anytime. You can roll back any file or folder to its exact state from 10 minutes, 10 hours or even weeks ago.
  • Local & Private: Your code stays on your machine inside a .wsn folder stored at the root user directory.
  • Noise Filtering: You can ignore files, and folders using a standard .wsnignore text file. That updates on save.

.wsnignore must be placed at the root directory of your project.

How it works

  1. Watch: WSN runs in the background to monitor your projects for changes.
  2. Manage: Use the WSNCmd command-line tool to recover your entire file tree or a single file at any specific time.
    You can also annotate milestones to track the history of your project.

How to use?

These are the commands you can use with WSNCmd.
You must run these commands from the root directory of your project:

Register: wsncmd --register [ProjectPath]
Unregister: wsncmd --unregister
Restore: wsncmd --restore [FileOrFolder] [UnixTimestamp]
Annotate: wsncmd --annotate [Message] [UnixTimestamp]
Unannotate: wsncmd --unannotate [OldMessage]

[ProjectPath] is the root directory of your project.
[FileOrFolder] ends with "/" if it is a folder. It accepts subfolders.
[UnixTimestamp] is a Unix Timestamp in seconds.
